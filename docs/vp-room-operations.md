# #副社長室 チャンネル 運用ルール

**作成:** 2026-06-04  
**管理:** 黒川 Chief of Staff  
**チャンネル種別:** 社内限定（外部非公開）

---

## 目的

ChatGPT Desktop（AI副社長）の判断・CEO外部相談結果・重要メモを  
AI_WORKER社内に橋渡しするためのバッファチャンネル。

CEO が毎回 Discord へ手動コピペする負担を減らす。

---

## 利用シーン

| 用途 | 例 |
|------|---|
| ChatGPT副社長の判断メモ | 「Phase2は延期が妥当」などの判断結果 |
| CEO外部相談の結果メモ | 顧問・パートナーとの相談サマリ |
| 方針変更の初期メモ | 「YouTube診断AIの方向を修正する」 |
| 緊急インシデント情報 | 「本番環境で障害発生」 |
| AI社員への確認依頼候補 | 「宮城Leadに技術的に可能か確認したい」 |

---

## 掲載ルール

### ✅ 掲載して良いもの
- ChatGPT Desktopから受け取った判断テキスト
- CEOが作成したメモ・議事録
- 内部方針の変更候補
- AI社員への依頼・相談候補

### ❌ 掲載禁止
- 外部顧客の個人情報（氏名・連絡先・住所）
- 契約金額・詳細な見積もり
- APIキー・パスワード・認証情報
- 外部公開前提のプレスリリース原稿

---

## Desktop Inbox Bridge との連携

### 運用フロー

```
CEO が ChatGPT Desktop で作業
        ↓
ChatGPT副社長の判断・メモを取得
        ↓
data/inbox/gpt/incoming.md にペースト
        ↓
Discord で !inbox check を実行
        ↓
黒川が分類・提案（実行は行わない）
        ↓
CEO が確認し、必要なコマンドを手動実行
```

### incoming.md の書き方

`data/inbox/gpt/incoming.md` に自由テキストをペースト。  
以下のようなセクション区切りを使うと分類精度が上がる（任意）:

```markdown
## 判断
○○の方針は△△に決定した。

## タスク
- !youtube diagnose の精度改善が必要
- 診断UIをWebアプリに追加する

## 確認依頼
守谷CTOに技術的実現性を確認したい

## 緊急
本番環境のAPIクォータが残り10%を切った
```

---

## セキュリティ

- このチャンネルは AI_WORKER 社内限定
- 外部公開・外部共有 禁止
- `data/inbox/gpt/` `data/outbox/gpt/` は `.gitignore` 対象
- 内容は `redact()` でマスキングされてから処理される
- ChatGPTメモを AI_WORKER 外部へ転送しない

---

## 黒川の役割と制限

**役割:**
- incoming.md の内容を分類する
- 実行候補コマンドを提案する
- 返信待ちレポートを管理する

**禁止:**
- 自動で `!task add` を実行しない
- 自動で `!decision log` を実行しない
- 自動で `!incident open` を実行しない
- 自動で `!msg send` を実行しない
- Git 操作しない
- 外部通信しない

> 黒川は「提案まで」。実行は必ず CEO が確認してから手動で行う。
