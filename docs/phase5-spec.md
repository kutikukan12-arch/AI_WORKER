## 概要

Phase4（タスク管理・ナイトバッチ・AI会議）に以下を追加。

---

## 追加された機能

### 1. タスクキュー（`bot/utils/task-queue.js`）

#### 概要

`!claude` コマンドが複数同時に送られた場合、タスクをキューに積んで
1件ずつ順番に処理する。

#### 動作

| 状況 | 動作 |
|------|------|
| 同時実行数以下 | 即座に実行開始 |
| 同時実行数超過 | キューに追加し「X番目待機」と通知 |
| キュータスク完了 | 次のタスクを自動で実行開始 |

#### 設定

```env
MAX_CONCURRENT_TASKS=1   同時実行数（デフォルト: 1）
```

#### コマンド

```
!queue              キュー状況を表示（実行中/待機中）
!queue clear        待機中のタスクをクリア（オーナーのみ）
```

---

### 2. 朝バッチ（`bot/utils/night-batch.js` 追加）

#### 概要

毎朝 `MORNING_BATCH_HOUR`（デフォルト: 8:00）に実行される自動バッチ。

#### 実行内容

| 処理 | 詳細 |
|------|------|
| 優先度再評価 | アクティブタスクをすべて priority.js で再スコアリング |
| 優先度変更通知 | スコアが変わったタスクを一覧表示 |
| 長期待機警告 | 24時間以上「人間確認待ち」のタスクを警告 |
| Discord 通知 | `MORNING_BATCH_CHANNEL_ID` または `BATCH_CHANNEL_ID` へ送信 |

#### 設定

```env
MORNING_BATCH_HOUR=8             実行時刻（時）
MORNING_BATCH_MINUTE=0           実行時刻（分）
MORNING_BATCH_CHANNEL_ID=        通知チャンネル（空=BATCH_CHANNEL_IDを使用）
```

---

### 3. 新コマンド一覧

| コマンド | 説明 |
|---------|------|
| `!queue` | **NEW** タスクキュー状況を表示 |
| `!queue clear` | **NEW** 待機中タスクをクリア（オーナーのみ） |
| `!claude <指示>` | タスクキューに追加して実行（Phase1から継続・強化） |
| `!task list` | タスク一覧（Phase4から継続） |
| `!meeting <議題>` | AI チーム会議（Phase4から継続） |
| `!batch` | ナイトバッチ手動実行（Phase4から継続） |

## 処理フロー（Phase5完全版）

```
!claude コマンド受信
  │
  ├─ セキュリティチェック
  │
  ├─ Phase5: タスクキューへ追加
  │   ├─ 空きあり → 即時実行
  │   └─ 満杯    → キューに追加 → 「X番目待機」を返信
  │
  ├─ STEP1: Claude Code 実行
  ├─ STEP2: AI レビュー
  ├─ STEP3: Codex 依頼生成
  ├─ STEP4: 自動フィードバック
  ├─ STEP5: GitHub Push / PR 作成
  ├─ STEP6: 次タスク判断
  └─ STEP7: 完了通知 → 次のキュータスクを自動実行

毎朝 MORNING_BATCH_HOUR 時
  → アクティブタスクの優先度再評価
  → 変更があれば通知
  → 24h以上待機タスクを警告
```

---

## リスクと対策

| リスク | 対策 |
|--------|------|
| キューが詰まる | `!queue clear` でオーナーが強制クリア可能 |
| 朝バッチと夜バッチが競合 | タイマーを分離（morningTimer / batchTimer） |
| 優先度の過剰変更 | priority.js のキーワードは変更せず再スコアのみ |
