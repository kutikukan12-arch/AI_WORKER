## 1. システム概要

AI_WORKER は Discord 上で動作する自律型 AI 開発 Bot。  
Discord からタスクを受け取り、Claude Code CLI を呼び出して実装・レビュー・GitHub push を自律的に実行する。

### 1.1 主要モジュール構成

```
Discord
  ↓ discord.js v14
bot/index.js（コマンドルーター・5,000行超）
  ├── claude-runner.js       Claude Code CLI (spawn)
  ├── codex.js               OpenAI GPT-4o レビュー（retry 3回）
  ├── github.js              git commit / push
  ├── github-pr.js           GitHub API PR 作成
  ├── task-manager.js        data/tasks.json (CRUD + Lease 30分)
  ├── task-queue.js          同時実行数制御
  ├── approval-manager.js    data/approvals.json（承認フロー）
  ├── auto-project-runner.js data/runner-state.json（自律ループ）
  ├── project-planner.js     LLM 計画生成（planProjectGoalsBest）
  ├── quality-gate.js        RED/YELLOW/GREEN 判定
  ├── auto-policy.js         タスク安全分類（4段階）
  ├── worker-registry.js     data/workers.json
  ├── project-manager.js     data/projects.json
  ├── company-manager.js     人員推奨・適用
  ├── night-batch.js         定期バッチ（02:00 / 08:00）
  ├── dashboard-server.js    Web 管理 UI（HTTP + WebSocket）
  └── youtube-predictor.js   YouTube 視聴予測 ML モデル
```

### 1.2 状態管理

## 3. 自律ループアーキテクチャ

### 3.1 `!project run` — 完全自律実行フロー

```
!project run <id>
  │
  ▼
PRE-RUN QA（品質チェック）
  │
  ▼
_runProjectLoop（ループ開始）
  ├── planProjectGoalsBest（LLM でタスク計画生成）
  ├── タスク取得 → Auto Policy 判定 → Claude Code 実行
  ├── 完了後フック: runPlannerStepAsync（FIX/REVIEW 自動生成）
  ├── [3タスクごと] MID-RUN Gate（YELLOW/RED → FIX 自動登録）
  └── HUMAN_CHECK 発火時 → awaiting_human 状態で停止
        ├── !approve → ループ再開
        └── !deny → 安全停止（_teardown）
  │
  ▼
POST-RUN QA（最終品質評価）
```

### 3.2 Auto Policy — タスク安全分類

| ポリシー | 条件 | 動作 |
|---------|------|------|
| `AUTO_SAFE` | DOCS/RESEARCH/TEST 系 | 自動実行 |
| `AI_REVIEW` | 通常の IMPLEMENT | Codex レビュー後実行 |
| `HUMAN_APPROVAL` | 高危険度タスク | 人間承認が必要 |
| `BLOCKED` | セキュリティ危険パターン | 実行禁止 |

### 3.3 Quality Gate

| 判定 | 条件 | 対応 |
|-----|------|------|
| GREEN | 全テスト通過・エラーなし | 継続 |
| YELLOW | 軽微な問題あり | 警告・継続 |
| RED | テスト失敗・重大エラー | FIX タスク自動生成・一時停止 |

soft RED 発生時は優先度「高」の FIX タスクを自動登録し、解決後に継続。

## 4. YouTube 投稿前予測 AI

### 4.1 概要

投稿前情報（タイトル・チャンネル名・タグ・登録者数等）から再生数を予測する ML モデル。  
モデル不在時はルールベースにフォールバックするため、クローン直後でも動作する。

### 4.2 主要ファイル

| ファイル | 役割 |
|---------|------|
| `bot/utils/youtube-predictor.js` | 予測エンジン（ロジスティック回帰 + ルールベース） |
| `bot/utils/youtube-feature-extractor.js` | 16次元特徴量エンコード |
| `bot/utils/youtube-data-collector.js` | 動画データ収集・管理 |
| `bot/utils/youtube-seed-presets.js` | シードデータ定義 |
| `bot/utils/ai-model-v2.js` | 汎用 ML モデル（FEATURE_DIM=26、YouTube 予測には使用不可）|
| `bot/predict-cli.js` | CLI インターフェース |

### 4.3 特徴量設計（16次元）

`youtube-feature-extractor.js` が以下を数値化する。

| 特徴量カテゴリ | 内容 |
|------------|------|
| タイトル特性 | 文字数・数字有無・感情語・キーワード |
| チャンネル規模 | 登録者数（対数正規化）|
| タグ品質 | タグ数・特定タグ含有 |
| 投稿タイミング | 曜日・時間帯 |
| ジャンル | VTuber/切り抜き/音楽/実写/ゲーム実況 |

投稿前入力（viewCount=0）で意味ある予測が返ること（**L-15 再発防止対応済み**）。

### 4.4 モデル動作仕様

```
入力: { title, channelName, tags, subs, genre, scheduledAt, ... }
  ↓
youtube-feature-extractor.js: 16次元ベクトル化
  ↓
youtube-predictor.js:
  ├── MLモデル（data/youtube-model.json）あり かつ 学習サンプル ≥ 20
  │     → ロジスティック回帰スコア（最大 70% 混合）+ ルールスコア（30%）
  └── MLモデルなし
        → ルールベーススコアのみ
  ↓
出力: { prediction: 'HIT'|'MISS'|'UNKNOWN', score, confidence, reasons }
```

### 4.5 `predict-cli.js` — CLI インターフェース

**npm install 不要**（外部依存ゼロ）。

```bash
# 基本使用
node bot/predict-cli.js --title "タイトル" --subs 50000

# 全オプション
node bot/predict-cli.js \
  --title "タイトル" \
  --channel "チャンネル名" \
  --tags "タグ1,タグ2" \
  --subs 50000 \
  --genre VTuber \
  --scheduled "2026-06-07T21:00:00+09:00"
```

**出力例:**
```
予測: HIT (スコア: 0.73, 信頼度: medium)
主要因: タイトル文字数が適切範囲 / 登録者数が多い / 感情語を含む
```

### 4.6 モデル学習

```bash
# Discord コマンド
!train

# または直接実行
node -e "require('./bot/utils/youtube-predictor').trainModel()"
```

学習データは `data/youtube-videos.json`（HIT/MISS ラベル付き動画データ）。  
MAPE・方向性正解率を学習完了通知に表示する。
