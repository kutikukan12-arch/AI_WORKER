# Natural CEO Interface — セットアップガイド

> 対象: AI_WORKER Bot 管理者 (CEO)
> 目的: CEO が自然な日本語で Bot を操作できる Natural CEO Interface を有効にする

---

## 概要

Natural CEO Interface を使うと、コマンドを覚えなくても自然文で AI_WORKER を操作できます。

| 自然文の例 | 動作 |
|---|---|
| 「今どう？」「YouTubeどう？」 | タスク進捗サマリーを表示 |
| 「進めて」「続けて」 | 次のタスクを実行 |
| 「問題ある？」「止まってない？」 | 問題・ブロッカーを確認 |
| 「今日なにした？」「今日どうだった？」 | 本日の作業サマリー |
| 「Bot動いてる？」「AI_WORKER動いてる？」 | Bot の稼働状況確認 |
| 「Bot再起動して」「再起動して」 | Bot を安全に再起動 |
| 「黒川起動して」「AI_WORKER起こして」 | 黒川CoS（Operator）を起動 |
| 「黒川止めて」「黒川停止して」 | 黒川CoS（Operator）を停止 |

また、承認カード（Approval Card）の「✅ 承認」「⏸ 待機」「🔍 詳細確認」ボタンも
CEO_USER_IDS が設定されている場合のみ操作できます。

---

## ⚠️ 設定が必要な理由

**CEO_USER_IDS が未設定の場合**、Bot は起動時に以下の警告を出力します：

```
⚠️ CEO_USER_IDS 未設定 — 自然文CEO操作は無効です
```

この状態では：
- 自然文（コマンド以外のメッセージ）は全て無視されます
- Approval Card のボタンを押しても「この操作はCEOのみ実行できます」と拒否されます

これは**意図的な fail-closed 設計**です。未設定 = 誰も操作できない（安全側）。

---

## セットアップ手順

### ステップ1: Discord のユーザー ID を取得する

1. Discord アプリを開く
2. **設定（歯車アイコン）→ 詳細設定 → 開発者モード を ON** にする
3. 自分のアカウントアイコンを **右クリック → 「IDをコピー」**
4. コピーした数字列（`123456789012345678` のような18桁）をメモする

> ⚠️ **チャンネルID ではなくユーザーID です**
> チャンネルを右クリックしても別のIDが取れます。必ず自分のアカウントを右クリックしてください。

---

### ステップ2: .env に CEO_USER_IDS を設定する

`.env` ファイルを開き、`Natural CEO Interface` セクションを探して入力します：

```env
# Natural CEO Interface
CEO_USER_IDS=123456789012345678
```

複数人を設定する場合（カンマ区切り）：

```env
CEO_USER_IDS=111111111111111111,222222222222222222
```

> ⚠️ **`.env` ファイルは Git にコミットしないでください**（.gitignore 対象）

---

### ステップ3: Bot を再起動する

```
!restart
```

または Discord の Bot 管理チャンネルで Bot プロセスを再起動してください。

---

### ステップ4: 動作確認する

Bot が起動したら、設定した CEO アカウントから以下を送信して確認：

```
Bot動いてる？
```

正常に設定されていれば Bot の稼働状況が返ってきます。

---

## 関連環境変数

| 変数名 | 説明 | 必須 |
|---|---|---|
| `CEO_USER_IDS` | CEO の Discord ユーザーID（カンマ区切り） | Natural CEO Interface 有効化に必須 |
| `DISCORD_OWNER_ID` | Bot オーナーの Discord ユーザーID（危険操作時メンション・Bot再起動の安全ゲート） | 推奨 |

> `CEO_USER_IDS` と `DISCORD_OWNER_ID` は通常同じ値を設定します。
> `DISCORD_OWNER_ID` は Bot の危険操作ゲートとして使われます。

---

## トラブルシューティング

### 「この操作はCEOのみ実行できます」と返ってくる

- `CEO_USER_IDS` が正しく設定されているか確認してください
- **チャンネルID ではなくユーザーID** を設定しているか確認してください
- Bot を再起動していない場合は再起動してください

### Bot が無反応（自然文を無視する）

- `CEO_USER_IDS` が未設定の場合、全ての自然文は無視されます
- Bot 起動ログに `⚠️ CEO_USER_IDS 未設定` が出ていないか確認してください
- 送信チャンネルが `ALLOWED_CHANNEL_IDS` に含まれているか確認してください

### 起動ログで確認する

正常に設定されている場合、起動ログに以下が出力されます：

```
[CEO] CEO_USER_IDS 設定済み (1件) — 自然文CEO操作: 有効
```

未設定の場合：

```
⚠️ CEO_USER_IDS 未設定 — 自然文CEO操作は無効です。.env に CEO_USER_IDS を設定して Bot を再起動してください。
```
