## 1. テスト方針

### 1.1 テスト種別

| 種別 | 対象 | YouTube API 実呼び出し |
|------|------|----------------------|
| **単体テスト** | 純粋関数（parseDuration, calcBuzzRatio, filterHit 等） | 不要 |
| **モックテスト** | API クライアント関数（ytSearch 等）、collectGenreSeeds | `fetch` をモック |
| **ファイルI/Oテスト** | saveSeeds / loadSeeds / クォータ管理 | 不要（tmp ディレクトリ使用） |
| **手動確認** | Discord コマンド連携、実データ収集 | **必要**（本物のAPIキー使用） |

### 1.2 テストフレームワーク

Node.js 組み込み `assert` モジュールのみ使用（Jest 不要）。  
プロジェクトの既存テスト（`tests/mask_secret_test.js` 等）と同パターン:

```javascript
function test(label, fn) { ... }
```

### 1.3 テスト対象関数一覧

| 関数 | テストケース ID | カテゴリ |
|------|--------------|----------|
| `parseDuration` | T-01 | 単体 |
| `calcBuzzRatio` | T-02 | 単体 |
| `filterHit` | T-03a | 単体 |
| `filterMiss` | T-03b | 単体 |
| `toSeedVideo` | T-04 | 単体 |
| `estimateQuotaCost` | T-05 | 単体 |
| `getSeedStats` | T-06 | ファイルI/O |
| `saveSeeds` / `loadSeeds` | T-07 | ファイルI/O |
| クォータ管理 | T-08 | ファイルI/O |
| APIキー未設定エラー | T-09 | モック |
| `ytSearch` モック | T-10 | モック |
| `collectGenreSeeds` 統合 | T-11 | モック |

### 2.2 seed-collector.js へのテスト用エクスポート追加仕様

Phase 2 の `bot/utils/seed-collector.js` 末尾に以下を追加する必要がある:

```javascript
// ─── テスト用エクスポート（本番では使用しない）─────────────
if (process.env.NODE_ENV === 'test') {
  module.exports._testExports = {
    parseDuration,
    calcBuzzRatio,
    filterHit,
    filterMiss,
    toSeedVideo,
    incrementQuotaCounter,
    getRemainingQuota,
  };

  let _seedsDirOverride = null;
  let _quotaLogPathOverride = null;

  module.exports._testHelpers = {
    setSeedsDir: (dir) => { _seedsDirOverride = dir; },
    setQuotaLogPath: (p) => { _quotaLogPathOverride = p; },
    saveSeeds: (genre, label, videos) => {
      const dir = path.join(_seedsDirOverride || SEEDS_DIR, genre, label);
      fs.mkdirSync(dir, { recursive: true });
      for (const v of videos) {
        fs.writeFileSync(path.join(dir, `${v.videoId}.json`), JSON.stringify(v, null, 2), 'utf-8');
      }
    },
  };
}
```

## 7. 既知の注意点・落とし穴

### 7.1 `statistics` フィールドが存在しない動画

YouTube API では `statistics` がプライベート設定の動画で取得できない場合がある。  
`toSeedVideo` では `|| '0'` フォールバックが必要（Phase 2 §2.4 参照）。

**テスト確認:**
```javascript
// statistics が undefined の場合クラッシュしないか
const noStats = { ...mockYtVideo, statistics: {} };
const result = toSeedVideo(noStats, mockChannel, 'VTuber', 'hit');
assert.strictEqual(result.viewCount, 0);  // フォールバック値
```

### 7.2 `hiddenSubscriberCount: true` のチャンネル

`calcBuzzRatio` は登録者数が取得できない場合 `null` を返す設計だが、  
`ytChannelsList` の戻り値で `subscriberCount` が存在しないケースを `toSeedVideo` で確認する必要がある。

**確認点:** `subscriberCount` が文字列 `"0"` なのか、フィールド自体が存在しないのかによって `parseInt` の結果が変わる。

### 7.3 ISO 8601 duration の edge case

- `P1D`（1日）のような hour/minute/second を含まない形式は実用上まれだが、  
  `parseDuration` でマッチしない場合 `0` を返すことを確認する。
- `PT0S` は有効な 0秒動画（YouTube にライブ配信開始直後の動画で見られる）。

### 7.4 Windows パス区切り

`SEEDS_DIR` に `path.join()` を使っているため Windows (`\`) / Linux (`/`) 両対応だが、  
テスト時に `tmpDir` を使う際も `path.join()` を使うこと（`/` ハードコード禁止）。

### 7.5 fetch のグローバル汚染

テストで `globalThis.fetch` をモック差し替えした場合、テスト後に元に戻すこと:

```javascript
const originalFetch = globalThis.fetch;
globalThis.fetch = mockFetch;
try {
  await runTest();
} finally {
  globalThis.fetch = originalFetch;
}
```
