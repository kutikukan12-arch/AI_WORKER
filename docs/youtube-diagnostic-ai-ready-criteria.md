# YouTube診断AI β — PM合格基準（READY条件）

- **作成:** 🅴 市川（PM）／ 2026-06-04
- **目的:** D1〜D4 修正の**前に**、商品としてのREADY条件を確定する。宮城実装後に「好み」で判断しないための、測定可能な合格ライン。
- **制約:** 新機能追加は禁止（サムネ診断/AI高度化/競合分析/課金は対象外）。**今ある診断を「人に渡せる状態」にすることだけ**を見る。
- **判定基準:** D11「AI商品の価値＝意思決定改善」／L22「投稿前診断はユーザーが今すぐ実行できる助言のみを出す」。
- **READYの大原則:** 各条件は **YES/NO で判定できる**こと。曖昧な「良い感じ」は不可。

---

## ① 初回30秒体験 READY条件

初見ユーザーが**説明なし**で、開いて30秒以内に次の3つを言えること。

| # | 伝わるべきこと | 合格条件（画面が満たすこと） |
|---|---|---|
| 1-a | これは何をするサービスか | 画面上部に**1行の価値文**がある（例：「投稿前に、動画の伸ばし方を診断します」）。「予測」「占い」表現は禁止 |
| 1-b | 何が改善できるのか | 入力前から、診断対象（タイトル/タグ/ジャンル/投稿時間）が見て分かる。入力欄のラベルが平易 |
| 1-c | 次に何をすればいいか | 必須は**タイトル＋ジャンルのみ**と分かり、診断ボタンが1つで明確。迷う選択肢を置かない |

**合格テスト:** 初見5人中4人以上が、説明なしで「何のサービスか／何を改善できるか／次に何を押すか」を30秒以内に言える。

## ② 改善提案の品質基準（D11 / L22）

> **改善提案は「投稿前に、このユーザーが、今すぐ実行できる行動」のみ。**

### 合格条件（すべて満たす）
- [ ] **2-a 全提案が実行可能アクション**：タイトル変更／タグ追加／説明文改善／投稿時間変更／構成・尺調整 のいずれか。
- [ ] **2-b 実行不能な構造要因を「改善提案」として出さない**：登録者を増やす／過去実績を伸ばす 等は提案欄に出さない（出すなら参考の文脈補足にとどめ、改善Topに置かない）。
- [ ] **2-c 改善提案は最低1件、必ず実行可能なものを返す**（提案が登録者数だけ、で終わらない）。
- [ ] **2-d タイトルが弱い時はタイトル改善を必ず含む**（最大レバーの取りこぼし禁止）。
- [ ] **2-e 各提案に「対策（どう直すか）」が具体的に付く**（「数字を入れる」等の方向＋例）。

### NG例（これが出たら不合格）
- 「登録者を増やしてください」だけで終わる
- 弱い汎用タイトル（例「歌ってみた」）にタイトル改善が出ない

## ③ 6軸表示基準

> 採点で不安にさせるのではなく、**各軸が「改善行動」に必ず接続**していること。

| 合格条件 | 内容 |
|---|---|
| 3-a 6軸を表示 | CTR適性/視聴維持/SEO/感情フック/投稿タイミング/競合差別化 を表示 |
| 3-b 低スコアは改善方向とセット | **裸の点数を出さない。** 低い軸は必ず「→ ○○で改善可能」を併記 |
| 3-c 前向きフレーミング | 「ダメ」ではなく「ここを直せば伸びる」の言い回し。ランク表記も萎縮させない |
| 3-d 免責の常時表示 | 「本診断は最適化度を示すもので、再生回数を予測・保証しません」 |

**表示の型（例）:**
```
悪い:  タイトル 40点
良い:  タイトル 40点 → キーワードを1つ追加すると改善可能
```

## ④ βテスト合格条件（相沢CS確認）

> 精度ではなく **「行動が取れたか・不安が減ったか・また使いたいか」** で合否を見る。

### 相沢CSが各被験者に確認する質問
| 確認 | 質問（中立・誘導しない） | 合格に数える条件 |
|---|---|---|
| V-1 次の行動 | 「診断を見て、次に何をしますか？」 | 具体的な改善行動を**自分の言葉で**言える |
| V-2 改善行動 | 「実際にタイトル/タグを直してみますか？」→可能なら直して再診断 | 1つ以上、改善行動を**実際に取った**（再診断で点が動くのを体験） |
| V-3 不安軽減 | 「やる前より、少し安心できましたか？」 | 「減った/安心した」方向の反応 |
| V-4 再利用 | 「次の投稿前にも、また使いそうですか？」 | 「使う」意向 |

### βテスト合格ライン（母数5〜6人）
- [ ] **V-1 次の行動が言える：80%以上**
- [ ] **V-2 改善行動を実際に取った：過半数**
- [ ] **V-3 不安が減った：過半数**
- [ ] **V-4 また使いたい：過半数**
- [ ] 禁止表現（再生数予測/レンジ/保証）の出現：**0件**

## β版 READY 総合ゲート（全項目 PASS で「人に渡せる」）

- [ ] ①初回30秒：5人中4人が説明なしで3点を言える
- [ ] ②改善提案：2-a〜2-e すべて満たす（実行可能アクションのみ・タイトル取りこぼしゼロ）
- [ ] ③6軸：裸の点数ゼロ・全軸が改善方向とセット・免責常時表示
- [ ] ④βテスト：V-1〜V-4 が上記ラインを満たす・禁止表現0件

> 1つでもNOなら**β未公開**。READYは「精度が高い」ではなく「**渡された人が、次の行動を取れて、不安が減って、また使いたい**」状態。

## 宮城Leadへの実装要求（READYを満たすために必要なこと・How は問わない）

> 新機能追加は禁止。**今ある診断出力を、上記基準を満たす形に整えるだけ。**

1. **改善提案フィルタ（最重要）**：実行不能な構造要因（登録者数/チャンネル規模/過去実績）を**改善提案から除外**。改善提案は実行可能アクションのみ。（②2-a/2-b/2-c）
2. **タイトル改善の取りこぼし防止**：弱い/汎用タイトル時はタイトル改善を必ず提案に含める。（②2-d）
3. **6軸の表示＋改善方向の併記**：各軸を表示し、低スコアは「→ ○○で改善可能」をセットで出す。裸の点数を出さない。（③3-a/3-b）
4. **免責の常時表示**。（③3-d）
5. **初回30秒UI**：1行の価値文・必須2項目・診断ボタン1つ（Web最小UI）。（①）
6. （引き続き）**禁止表現ゼロ**：再生数の数値/レンジ/断定語を出さない。

> ※ いずれも「今ある診断ロジックの出力整形・提案の取捨」であり、AI高度化・新分析機能ではない。診断エンジンの本質仕様は変えない。

## 相沢CSへの確認項目（再掲・βテスト用）

- V-1 次の行動が言えるか／V-2 改善行動を実際に取ったか／V-3 不安が減ったか／V-4 また使いたいか（§④の質問と合格ラインを使用）
