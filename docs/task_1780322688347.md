# YouTube 予測 AI 使用ガイド

- **作成日:** 2026-06-01
- **対象ファイル:** `bot/utils/youtube-predictor.js`, `bot/utils/youtube-feature-extractor.js`, `bot/utils/youtube-data-collector.js`, `bot/utils/youtube-api-client.js`
- **関連仕様書:** [再生回数予測AI MVP仕様書](./youtube-prediction-ai-mvp-spec.md)

---

## 概要

Discord コマンド経由で YouTube 動画の「ヒット / ミス」を予測するシステム。  
**ルールベーススコア**と**機械学習モデル（ロジスティック回帰）**を混合して確率を算出する。

---

## Discord コマンド一覧

| コマンド | 権限 | 説明 |
|---------|------|------|
| `!youtube predict <URL>` | 全員 | 動画 URL を渡してヒット予測を実行 |
| `!youtube status` | 全員 | API クォータとモデルの状態を表示 |
| `!youtube collect <genre> <query>` | 管理者 | シードデータを YouTube API から収集 |
| `!youtube train` | 管理者 | 収集済みデータでモデルを訓練 |

---

## 基本的な使い方

### 1. 予測を試す

```
!youtube predict https://www.youtube.com/watch?v=xxxxxxxxxx
```

出力例：
```
🎬 YouTube ヒット予測

🟢 確率: 74%  `hit`  (信頼度: medium)
📊 buzz_ratio: 6.23  (hit>5.0 / miss<0.3)
🤖 ML モデル使用 (学習数: 52件)

📺 【歌ってみた】〇〇〇〇
👁️ 再生数: 1,234,567  👥 登録者: 198,000
⏱️ 長さ: 4分32秒  🏷️ タグ: 12個
```

### 2. 予測結果の見方

| ラベル | 条件 | 意味 |
|--------|------|------|
| `hit` | 確率 ≥ 60% | 登録者数の 5 倍以上の再生が付く可能性 |
| `miss` | 確率 ≤ 40% | 登録者数の 0.3 倍未満の再生になる可能性 |
| `unknown` | 40% < 確率 < 60% | 中間帯のため判定が難しい |

**信頼度の意味:**

| 信頼度 | 条件 |
|--------|------|
| `low` | ML モデルの学習データが 20 件未満（ルールベース予測のみ） |
| `medium` | ML モデルが利用可能（学習データ 20 件以上） |

## 初回セットアップ手順

ML モデルを使うためには以下の順序でセットアップが必要。

### Step 1: 環境変数の設定

`.env` に YouTube Data API v3 のキーを追加する：

```
YOUTUBE_API_KEY=AIzaSy...
```

### Step 2: シードデータの収集

```
!youtube collect vtuber VTuber 歌ってみた
!youtube collect gaming ゲーム実況 神回
```

- `<genre>` はファイル名になるため英数字・ハイフン推奨
- 1 回の収集で hit 最大 100 件 + miss 最大 20 件を取得
- クォータを約 200〜600 units 消費する（1 日 10,000 units 上限）

### Step 3: モデル訓練

```
!youtube train
```

- 全ジャンルの収集データを合算して訓練する
- 最低 10 件のサンプルが必要
- ML が有効になるのは 20 件以上から

### Step 4: ステータス確認

```
!youtube status
```

## クォータ管理

YouTube Data API v3 は **1 日 10,000 units** の制限がある。

| API 呼び出し | コスト |
|------------|--------|
| `search.list`（動画検索） | 100 units / 回 |
| `videos.list`（動画詳細） | 1 unit / 50 件 |
| `channels.list`（チャンネル情報） | 1 unit / 回 |
| `playlistItems.list`（投稿一覧） | 1 unit / ページ |

**`!youtube status` で当日の使用量を常に確認すること。**  
クォータを使い切ると当日中はデータ収集・予測が不可能になる。

## 予測アルゴリズムの仕組み

### buzz_ratio（ルールベース）

```
buzz_ratio = viewCount / subscriberCount
```

| buzz_ratio | ラベル | スコア |
|-----------|--------|--------|
| > 5.0 | hit | 0.9 |
| < 0.3 | miss | 0.1 |
| その他 | - | 線形補間（0.1〜0.9） |

登録者数が非公開のチャンネルは buzz_ratio を計算できないためスコア 0.5 固定。

### ML モデル（ロジスティック回帰）

16 次元の特徴量ベクトルを使うロジスティック回帰。訓練データが 20 件以上集まると有効になる。

| 特徴量 | 内容 |
|--------|------|
| `like_ratio` | いいね数 / 再生数 |
| `comment_ratio` | コメント数 / 再生数 × 10 |
| `title_len_norm` | タイトル文字数 / 100 |
| `title_has_excl` | タイトルに「!」「！」を含むか |
| `title_has_quest` | タイトルに「?」「？」を含むか |
| `title_emoji_norm` | タイトルの絵文字数 / 5 |
| `title_caps_ratio` | タイトル内の全大文字単語の割合 |
| `tag_count_norm` | タグ数 / 30 |
| `desc_len_norm` | 説明文文字数 / 2000 |
| `duration_norm` | 動画尺（秒）/ 3600 |
| `published_hour_sin/cos` | 投稿時刻の周期エンコード |
| `published_dow_sin/cos` | 投稿曜日の周期エンコード |
| `sub_magnitude_norm` | log10(登録者数 + 1) / 8 |
| `bias` | 常に 1.0 |

### 混合スコア計算

```
mlWeight = min(mlSamples / 100, 0.7)
probability = mlScore × mlWeight + ruleScore × (1 - mlWeight)
```

- ML データ 20 件未満 → ルールベース 100%
- ML データ 20〜100 件 → ML の比率が段階的に増加
- ML データ 100 件以上 → ML 70% + ルール 30% で固定

## トラブルシューティング

### 予測が `unknown` ばかりになる

buzz_ratio が中間帯（0.3〜5.0）の動画に対してはルールベースも判定が難しい。  
ML モデルの学習データを増やすと精度が上がる。

### `クォータ超過` エラーが出る

当日のクォータを使い切った状態。翌日 0:00 UTC（日本時間 9:00）にリセットされるまで待つ。

### `登録者数非公開` エラーが出る

該当チャンネルは登録者数を非公開設定にしているため buzz_ratio を計算できない。  
collect コマンドではそのチャンネルをスキップし、predict コマンドではスコア 0.5 として扱う。

### `訓練サンプル不足` と表示される

`!youtube collect` で追加データを収集してから再度 `!youtube train` を実行する。  
hit と miss 合わせて最低 10 件必要。

### モデルが古い

`!youtube collect` で新しいデータを追加してから `!youtube train` を再実行することでモデルが更新される。  
`data/youtube-model.json` の `trainedAt` フィールドで最終訓練日時を確認できる。
