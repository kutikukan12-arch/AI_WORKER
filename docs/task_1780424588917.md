## 1. セットアップ方法

```bash
# 依存パッケージのインストール（プロジェクトルートで実行）
npm install

# .env ファイルが存在することを確認
ls .env
```

`.env` が存在しない場合は `.env.example` をコピーして作成してください。

## 2. YouTube API Key 設定方法

### API Key の取得手順

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセス
2. プロジェクトを選択（または新規作成）
3. 「APIとサービス」→「認証情報」→「認証情報を作成」→「API キー」
4. 「APIとサービス」→「ライブラリ」で **YouTube Data API v3** を有効化

### .env への設定

```dotenv
# .env ファイルの該当行を編集
YOUTUBE_API_KEY=AIzaSy...（取得した API キーを貼り付け）
```

### 設定確認

```bash
node -e "require('dotenv').config(); console.log(process.env.YOUTUBE_API_KEY ? 'OK' : 'NG')"
```

## 4. predict 使用例

### 投稿前予測（viewCount 未設定）

タイトル・タグ・尺・登録者数のみで予測します。buzz_ratio は使用されません。

```bash
# 基本
node bot/predict-cli.js -t "【衝撃】AIが仕事を奪う未来" -s 50000

# タグ・尺付き
node bot/predict-cli.js \
  -t "【衝撃】AIが仕事を奪う未来" \
  -s 50000 \
  --tags "AI,テクノロジー,未来" \
  -d 600

# チャンネル名・概要欄付き
node bot/predict-cli.js \
  -t "【初心者向け】Python入門 #1" \
  -c "テック解説チャンネル" \
  -s 12000 \
  --tags "Python,プログラミング,入門" \
  -d 900 \
  --desc "Pythonの基礎をわかりやすく解説します"
```

### 投稿後評価（viewCount あり）

実際の視聴数・いいね数からバズ率（buzz_ratio）を算出し、過去の類似動画実績と比較します。

```bash
# 投稿後の動画を評価
node bot/predict-cli.js \
  -t "daily vlog" \
  -s 1000 \
  -v 8500 \
  -l 300 \
  --comments 40 \
  -d 900

# JSON 出力付き（詳細な数値確認用）
node bot/predict-cli.js -t "料理動画" -s 5000 -v 15000 -l 800 --json
```

### 出力の見方

```
チャンネル : テック解説チャンネル
タイトル   : 【衝撃】AIが仕事を奪う未来
タグ       : AI, テクノロジー, 未来
登録者数   : 50,000
モード     : 投稿前予測（viewCount 未設定）

予測スコア : 67%（中程度のヒット可能性）
信頼度     : medium
予測再生数 : 2.5万〜12万回（中央値 5万回）
```

| 項目 | 説明 |
|------|------|
| 予測スコア | 0〜100%。ヒット確率の推定値 |
| 信頼度 | `high` / `medium` / `low`。学習データ量に依存 |
| 予測再生数 | 登録者数が設定されている場合のみ表示 |

## 5. collect — データ収集

`bot/utils/youtube-data-collector.js` が提供する収集機能です。YouTube Data API v3 を使い、ジャンル別のヒット動画・ミス動画のデータを `data/youtube-seeds/` に保存します。

### 収集対象ジャンル（`youtube-seed-presets.js`）

| ジャンルキー | ラベル |
|------------|--------|
| `vtuber` | VTuber・配信 |
| `gaming` | ゲーム実況 |
| `cooking` | 料理・グルメ |
| 他 | seed-presets.js を参照 |

### 収集フロー

1. `collectHitVideos(client, query)` — `search.list(order=viewCount)` で高再生数動画を収集
   - 投稿から 30 日以上経過した動画のみ対象（再生数が安定したもの）
   - `buzz_ratio > 5.0` をヒットと判定
2. `collectMissFromChannels(client, channelIds)` — ヒット動画のチャンネルのアップロード一覧から miss 動画を収集
   - `buzz_ratio < 0.3` をミスと判定

### buzz_ratio の定義

```
buzz_ratio = viewCount / subscriberCount
```

| 値 | ラベル |
|----|-------|
| > 5.0 | `hit`（バズ動画） |
| < 0.3 | `miss`（低迷動画） |
| その他 | ラベルなし（学習に使用されない） |

## 6. train — モデル学習

`bot/utils/youtube-predictor.js` が収集データからロジスティック回帰モデルをトレーニングします。学習済みウェイトは `data/youtube-model.json` に保存されます。

### 学習アルゴリズム

- **手法:** ロジスティック回帰（全バッチ SGD）
- **特徴量次元:** 16次元（`youtube-feature-extractor.js` の `FEATURE_DIM`）
- **学習率:** 0.05、L2正則化: 0.001、エポック: 400
- **最低サンプル数:** 20件（`MIN_ML_SAMPLES`）以上でMLモデルが有効化

### ML とルールベースの混合

```
最終スコア = ML スコア × mlWeight + ルールスコア × (1 - mlWeight)
mlWeight   = min(sampleCount / 100, 0.7)
```

- サンプルが少ない（< 20件）: ルールベースのみ使用
- サンプルが多い（≥ 100件）: ML が 70%、ルールが 30%

### 投稿前モードの特殊処理

投稿前（`viewCount = 0`）の場合、`like_ratio` など viewCount 依存の特徴量が 0 になり ML が miss 方向にバイアスするため、**ルールベースのみ**を使用します。

## 7. よくあるエラーと対処法

### `入力エラー: --title は必須です`

**原因:** `--title`（`-t`）オプションが指定されていない。

```bash
# 修正例
node bot/predict-cli.js --title "タイトルを入力してください"
```

### `--subs には数値を指定してください`

**原因:** 数値オプションに文字列が渡されている。

```bash
# NG
node bot/predict-cli.js -t "タイトル" -s "1万"

# OK
node bot/predict-cli.js -t "タイトル" -s 10000
```

### `YouTube API キー認証エラー（403）`

**原因:** `.env` の `YOUTUBE_API_KEY` が未設定または無効。

**対処法:**
1. `.env` ファイルを開き `YOUTUBE_API_KEY=` に正しいキーを設定
2. Google Cloud Console で YouTube Data API v3 が有効化されているか確認
3. API キーに IP 制限が設定されている場合はそれを確認

### `YouTube API クォータ上限超過（403）`

**原因:** 当日の API 使用量が 10,000 units を超えた。

**対処法:** 翌日以降に再試行してください（quota は毎日太平洋時間 0:00 にリセット）。

### `YouTube API リソース未発見（404）`

**原因:** 動画 ID またはチャンネル ID が無効。

**対処法:** ID に誤りがないか確認してください。削除済み動画は取得不可。

### `エラー（予測処理失敗）`

**原因:** 予測処理内の例外（特徴量エンコード失敗等）。

**対処法:** 入力値（特に `--duration`、`--subs`）が適切な数値か確認してください。`-h` でヘルプを参照。

## 8. API クォータ注意事項

YouTube Data API v3 の日次クォータ上限は **10,000 units/日** です。

### 各エンドポイントのコスト

| API | コスト | 用途 |
|-----|--------|------|
| `search.list` | 100 units/回 | キーワード検索（最大コスト） |
| `videos.list` | 1 unit/50件 | 動画詳細取得 |
| `channels.list` | 1 unit/50件 | チャンネル情報取得 |
| `playlistItems.list` | 1 unit/ページ | アップロード一覧取得 |

### 1ジャンル収集のコスト概算

```
search.list  × 2 pages = 200 units
videos.list  × 2 calls =   2 units
channels.list × 1 call =   1 unit
playlistItems × 3 pages =   3 units
─────────────────────────────────
合計 ≈ 206 units/クエリ
```

5ジャンル × 3クエリ = 15クエリ ≈ **3,090 units**（日次上限の約 31%）

### クォータ管理

使用量は `data/youtube-quota.json` に自動記録されます（日次リセット対応）。

```json
{ "date": "2026-06-03", "used": 206 }
```

### 注意点

- `predict-cli.js` 単体では YouTube API を呼び出しません（クォータ消費なし）
- クォータを消費するのはデータ収集（collect）処理のみ
- 本番環境と開発環境で API キーを分けることを推奨

## 9. 予測精度の限界

### 統計的限界

- 予測は「確率」であり断言ではありません。`予測スコア 67%` は「同条件の動画の約 67% がヒットした」という過去の傾向です
- 学習データが少ない場合（< 20件）はルールベースのみで動作し、精度は参考値程度です

### 考慮されない要因

| 要因 | 理由 |
|------|------|
| サムネイル品質 | 画像解析未実装 |
| 投稿タイミング | publishedAt は部分的に使用のみ |
| プロモーション費用 | データなし |
| 炎上・バズイベント | 予測不能 |
| アルゴリズム変動 | リアルタイム追従不可 |
| チャンネルの伸び率 | 登録者数の増加速度は未使用 |

### 信頼度の目安

| 信頼度 | 予測レンジ幅 | 状況 |
|--------|------------|------|
| `high` | 中央値 ± 30% | 学習データが十分 |
| `medium` | 中央値 ± 60% | 学習データが中程度 |
| `low` | 中央値 ± 100% | 学習データ不足またはルールベースのみ |

### 推奨使い方

- **A/Bテスト的活用:** 複数のタイトル案を比較して相対的に優れた方を選ぶ用途に適しています
- **過度な信頼は禁物:** 予測スコアが高くても実際にヒットしないケースは多数あります
- **データ蓄積で精度向上:** collect → train を繰り返すことでチャンネル固有の傾向を学習できます
