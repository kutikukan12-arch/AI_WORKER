## 1. 概要と注意事項

YouTube 動画の「ヒット / ミス」を予測するシステム。**ルールベーススコア（buzz_ratio）** と **ML モデル（ロジスティック回帰）** を混合して確率を算出する。

**重要な前提：**

- 予測は「過去の類似データとの比較」に基づく確率であり、再生回数の断言ではない
- ML モデルが有効になるにはシードデータの収集（`!youtube collect`）とモデル訓練（`!youtube train`）が必要
- YouTube Data API v3 のキー（`YOUTUBE_API_KEY`）が `.env` に設定されていないとデータ収集・URL 予測ができない

## 2. Discord コマンド一覧

| コマンド | 権限 | 説明 |
|---------|------|------|
| `!youtube predict <URL>` | 全員 | 投稿済み動画 URL をヒット予測 |
| `!youtube predict title="..." [options]` | 全員 | 投稿前メタデータで予測 |
| `!youtube status` | 全員 | API クォータとモデルの状態を確認 |
| `!youtube collect <genre> <query>` | 管理者 | 1 クエリ分のシードデータを収集 |
| `!youtube bulk-collect <genre>` | 管理者 | プリセットクエリで数百件を一括収集 |
| `!youtube train` | 管理者 | 収集済みデータでモデルを訓練 |

## 3. 予測の使い方（predict）

### 3.1 投稿済み動画 URL で予測

```
!youtube predict https://www.youtube.com/watch?v=XXXXXXXXXXX
```

YouTube Data API から動画の実績データを取得して予測する。`YOUTUBE_API_KEY` が必要。

**出力例：**

```
🎬 YouTube ヒット予測

🟢 確率: 74%  `hit`  (信頼度: medium)
📊 buzz_ratio: 6.23  (hit>5.0 / miss<0.3)
🤖 ML モデル使用 (学習数: 52件)

📺 【歌ってみた】〇〇〇〇
👁️ 再生数: 1,234,567  👥 登録者: 198,000
⏱️ 長さ: 4分32秒  🏷️ タグ: 12個
```

### 3.2 投稿前メタデータで予測

```
!youtube predict title="新曲歌ってみた！" tags="歌ってみた,vtuber" sec=300 subs=10000
```

**使用可能パラメータ：**

| パラメータ | 必須 | 説明 |
|-----------|------|------|
| `title="..."` | ★必須 | 動画タイトル |
| `subs=数値` | 推奨 | チャンネル登録者数（未指定は 0 扱い） |
| `tags="tag1,tag2"` | 任意 | タグ（カンマ区切り） |
| `sec=数値` | 任意 | 動画尺（秒）。例: 600 = 10 分 |
| `genre=キー` | 任意 | ジャンルキー（`vtuber`, `gaming` 等） |

**使用例：**

```
!youtube predict title="【衝撃】AIが仕事を奪う未来" subs=50000 tags="AI,テクノロジー" sec=600
```

## 4. データ収集の手順（collect / bulk-collect）

### 4.1 1 クエリ収集

```
!youtube collect <genre> <検索クエリ>
```

```
例:
!youtube collect vtuber VTuber 歌ってみた
!youtube collect gaming ゲーム実況 神回
```

- 1 回の収集で hit 最大 100 件 + miss 最大 20 件を取得
- クォータを約 200〜600 units 消費する

### 4.2 プリセット一括収集（推奨）

```
!youtube bulk-collect <genre>
```

```
例:
!youtube bulk-collect vtuber
!youtube bulk-collect gaming
```

- プリセットに登録された複数クエリを順番に実行し、数百件規模のシードを一括収集する
- `!youtube collect` の一括実行版。通常はこちらを使う
- 利用可能なジャンルキーは `!youtube bulk-collect` を引数なしで実行すると一覧表示される

**出力例：**

```
✅ 一括収集完了

ジャンル: `vtuber` (VTuber・配信)

クエリ結果:
[1] "VTuber 歌ってみた" → hit 97件
[2] "VTuber 雑談配信 切り抜き" → hit 82件
[3] "にじさんじ ホロライブ 動画" → hit 91件

累計保存:
🟢 hit: 270件  🔴 miss: 87件
📡 クォータ消費: 830 / 10,000 units

次のステップ: `!youtube train` でモデルを訓練してください
```

## 5. モデル訓練（train）

```
!youtube train
```

- 全ジャンルの収集データを合算して訓練する
- 最低 10 件のサンプルが必要（hit + miss 合計）
- ML が有効になるのは **20 件以上** から
- 訓練済みモデルは `data/youtube-model.json` に保存される

**訓練完了出力例：**

```
✅ YouTube 予測モデル 訓練完了

📊 サンプル数: 357件
🟢 hit: 270件  🔴 miss: 87件

モデルは `data/youtube-model.json` に保存されました。
`!youtube predict <URL>` で予測を試してください
```

## 7. CLI での使い方

Discord を使わずにターミナルから直接予測できる。

```
node bot/predict-cli.js --title "タイトル" --subs 10000 [options]
```

**オプション：**

| オプション | 省略形 | 説明 |
|-----------|-------|------|
| `--title` | `-t` | 動画タイトル（★必須） |
| `--channel` | `-c` | チャンネル名（表示用のみ） |
| `--subs` | `-s` | チャンネル登録者数（整数） |
| `--views` | `-v` | 視聴数（0 または省略 → 投稿前扱い） |
| `--likes` | `-l` | いいね数 |
| `--comments` | — | コメント数 |
| `--duration` | `-d` | 動画長（秒）。例: 600 = 10 分 |
| `--tags` | — | タグ（カンマ区切り）。例: `"AI,技術,解説"` |
| `--tag` | — | タグ（複数指定）。例: `--tag AI --tag 技術` |
| `--desc` | — | 概要欄テキスト |
| `--published` | — | 投稿日時 ISO 8601。例: `2024-03-01T18:00:00Z` |
| `--json` | — | 結果を JSON でも出力 |
| `--help` | `-h` | ヘルプ表示 |

**使用例（投稿前）：**

```bash
node bot/predict-cli.js -t "【衝撃】AIが仕事を奪う未来" -s 50000 --tags "AI,テクノロジー,未来" -d 600
```

**使用例（投稿済み）：**

```bash
node bot/predict-cli.js -t "daily vlog" -s 1000 -v 8500 -l 300 --comments 40 -d 900
```

## 8. 初回セットアップ手順

ML モデルを有効にするには以下の順序で実施する。

### Step 1: 環境変数の設定

`.env` に YouTube Data API v3 のキーを追加する：

```
YOUTUBE_API_KEY=AIzaSy...
```

### Step 2: シードデータの収集

```
!youtube bulk-collect vtuber
!youtube bulk-collect gaming
```

対応している全ジャンル分実施することで精度が向上する。

### Step 3: モデル訓練

```
!youtube train
```

### Step 4: 状態確認

```
!youtube status
```

`sampleCount` が 20 以上、`usedML=true` であれば ML モデルが有効な状態。

## 9. 予測アルゴリズムの仕組み

### 9.1 buzz_ratio（ルールベース）

```
buzz_ratio = viewCount / subscriberCount
```

| buzz_ratio | ラベル | スコア |
|-----------|--------|--------|
| > 5.0 | hit | 0.9 |
| < 0.3 | miss | 0.1 |
| 0.3〜5.0 | — | 線形補間（0.1〜0.9） |

登録者数が非公開のチャンネルは buzz_ratio を計算できないためスコア 0.5 固定。

### 9.2 ML モデル（ロジスティック回帰）

16 次元の特徴量ベクトルを使うロジスティック回帰。訓練データが 20 件以上集まると有効になる。

| 特徴量 | 内容 |
|--------|------|
| `like_ratio` | いいね数 / 再生数 |
| `comment_ratio` | コメント数 / 再生数 × 10 |
| `title_len_norm` | タイトル文字数 / 100 |
| `title_has_excl` | タイトルに「!」「！」を含むか |
| `title_has_quest` | タイトルに「?」「？」を含むか |
| `title_emoji_norm` | タイトルの絵文字数 / 5 |
| `title_caps_ratio` | タイトル内の全大文字単語の割合 |
| `tag_count_norm` | タグ数 / 30 |
| `desc_len_norm` | 説明文文字数 / 2000 |
| `duration_norm` | 動画尺（秒）/ 3600 |
| `published_hour_sin/cos` | 投稿時刻の周期エンコード（2次元） |
| `published_dow_sin/cos` | 投稿曜日の周期エンコード（2次元） |
| `sub_magnitude_norm` | log10(登録者数 + 1) / 8 |
| `bias` | 常に 1.0 |

### 9.3 混合スコア計算

```
mlWeight = min(mlSamples / 100, 0.7)
probability = mlScore × mlWeight + ruleScore × (1 - mlWeight)
```

| ML データ量 | 動作 |
|-------------|------|
| 20 件未満 | ルールベース 100% |
| 20〜100 件 | ML の比率が段階的に増加（最大 70%） |
| 100 件以上 | ML 70% + ルール 30% で固定 |

## 11. クォータ管理

YouTube Data API v3 は **1 日 10,000 units** の制限がある。

| API 呼び出し | コスト |
|------------|--------|
| `search.list`（動画検索） | 100 units / 回 |
| `videos.list`（動画詳細） | 1 unit / 50 件 |
| `channels.list`（チャンネル情報） | 1 unit / 50 件 |
| `playlistItems.list`（投稿一覧） | 1 unit / ページ |

**目安：**

- 1 クエリ収集（`!youtube collect`）: ≈ 200〜600 units
- 1 ジャンル一括収集（`!youtube bulk-collect`、3 クエリ）: ≈ 620〜1,800 units
- 5 ジャンル全収集: ≈ 3,100〜9,000 units（1 日で完結可能）
- `!youtube predict <URL>` 1 回: ≈ 3 units

**クォータ切れの対処：** 翌日 0:00 UTC（日本時間 9:00）に自動リセット。`!youtube status` で当日の使用量を常に確認すること。

## 12. 結果の読み方

### 予測ラベルと確率

| ラベル | 条件 | 意味 |
|--------|------|------|
| `hit` | 確率 ≥ 60% | 登録者数の 5 倍以上の再生が付く可能性が高い |
| `miss` | 確率 ≤ 40% | 登録者数の 0.3 倍未満の再生になる可能性が高い |
| `unknown` | 40% < 確率 < 60% | 中間帯のため判定が難しい |

### 信頼度

| 信頼度 | 条件 |
|--------|------|
| `low` | ML モデルの学習データが 20 件未満（ルールベース予測のみ） |
| `medium` | ML モデルが利用可能（学習データ 20 件以上） |

信頼度が `low` の場合は `!youtube collect` でデータを収集して `!youtube train` を実行することで精度が向上する。

## 13. トラブルシューティング

### 予測が `unknown` ばかりになる

buzz_ratio が中間帯（0.3〜5.0）の動画に対してはルールベースも判定が難しい。ML モデルの学習データを増やすことで精度が向上する。

### `クォータ超過` エラーが出る

当日のクォータを使い切った状態。翌日 0:00 UTC（日本時間 9:00）まで待つ。`!youtube status` で使用量を確認できる。

### `登録者数非公開` エラーが出る

該当チャンネルは登録者数を非公開設定にしているため buzz_ratio を計算できない。`collect` ではそのチャンネルをスキップし、`predict` ではスコア 0.5 として扱う。

### `訓練サンプル不足` と表示される

`!youtube collect` または `!youtube bulk-collect` で追加データを収集してから再度 `!youtube train` を実行する。hit + miss 合わせて最低 10 件必要。

### モデルが古い / 精度が低い

`!youtube collect` で新しいデータを追加してから `!youtube train` を再実行するとモデルが更新される。`data/youtube-model.json` の `trainedAt` フィールドで最終訓練日時を確認できる。

### `YOUTUBE_API_KEY が未設定` と表示される

`.env` ファイルに `YOUTUBE_API_KEY=AIzaSy...` を追加してからボットを再起動する。API キーなしでも投稿前メタデータ予測（`title="..."` 形式）は実行可能。

## 14. 制限事項と免責

- 予測結果は過去データに基づく確率であり、実際の再生回数を保証するものではない
- YouTube アルゴリズムの変動・トレンド・外部イベントは考慮されない
- 登録者数が非公開のチャンネルは buzz_ratio 計算不可（スコア 0.5 固定）
- シードデータが少ない状態（`low` 信頼度）の予測は参考程度に留める
- YouTube Data API v3 の利用規約に従い、収集データの第三者への再配布は禁止
