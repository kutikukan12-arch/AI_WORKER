## 要件

第三者がREADMEを読んで5分以内にCLIで投稿前予測を試せること。

## 調査結果

`bot/predict-cli.js` の依存関係を調査した結果、以下が判明した：

| モジュール | 種別 |
|-----------|------|
| `fs`, `path` | Node.js 組み込み |
| `./utils/youtube-predictor` | ローカル |
| `./utils/logger` | ローカル（fs/path のみ使用） |
| `./utils/youtube-feature-extractor` | ローカル（外部依存なし） |
| `./utils/youtube-data-collector` | ローカル（fs/path/logger のみ） |

**外部npm依存ゼロ** — `npm install` 不要でCLIが動作する。

また `youtube-predictor.js` はモデルファイル（`data/youtube-model.json`）不在時もルールベースにフォールバックするため、クローン直後でも予測が動作する。

## 変更内容

### README.md — 「5分でお試し」セクション改善

**変更前の問題点：**
1. `npm install` が必須ステップとして記載されていた（実際は不要）
2. `npm install` が遅い環境では5分制限内に収まらないリスクがあった
3. 投稿後モードの説明が不足していた

**変更後：**
1. セクション冒頭に「APIキー不要 / .env 設定不要 / npm install 不要」を明記
2. 手順を3ステップ→2ステップに削減（`npm install` を除外）
3. ヘルプには存在するが表中に欠けていた `-c / --channel` オプションを追加
4. 投稿後モードの使用例も末尾に追記

**結果として達成した5分フロー：**
