## task-manager.js

**役割:** タスクの作成・状態管理・リース制御・フォーマット出力を担う中核モジュール。  
**保存先:** `data/tasks.json`（現在タスク）、`data/history/YYYY-MM.json`（月次アーカイブ）

### 定数

| 定数 | 値 | 説明 |
|------|-----|------|
| `STATES` | `{ PENDING:'未着手', IN_PROGRESS:'作業中', REVIEWING:'レビュー待ち', AWAITING:'人間確認待ち', DONE:'完了', ON_HOLD:'保留' }` | タスク状態 |
| `TASK_TYPES` | `{ IMPLEMENT, FIX, REFACTOR, RESEARCH, DOCS, TEST, REVIEW }` | タスクタイプ |
| `TASK_SIZES` | `{ SMALL, MEDIUM, LARGE }` | タスクサイズ |
| `LEASE_TTL_MS` | `30 * 60 * 1000`（30分） | リース有効期間 |
| `STATE_EMOJI` | 状態→絵文字マップ | Discord 表示用 |
| `TYPE_EMOJI` | タイプ→絵文字マップ | Discord 表示用 |
| `SIZE_EMOJI` | サイズ→絵文字マップ | Discord 表示用 |

---

### `normalizeTaskType(type)`

タスクタイプ文字列を正規化する。

**引数**

| 名前 | 型 | 説明 |
|------|----|------|
| `type` | `string \| null` | タスクタイプ文字列（大文字小文字不問） |

**戻り値:** `string | null` — 有効な場合は大文字の type 文字列、無効・空・null の場合は `null`

**例**
```js
normalizeTaskType('fix')   // → 'FIX'
normalizeTaskType('bad')   // → null
normalizeTaskType(null)    // → null
```

---

### `estimateTaskSize(prompt)`

prompt の内容からタスクサイズを推定する。

**引数**

| 名前 | 型 | 説明 |
|------|----|------|
| `prompt` | `string` | タスクの依頼文 |

**戻り値:** `'SMALL' | 'MEDIUM' | 'LARGE'`

**判定基準**
- `LARGE` — 「全部」「一括」「全体的」等のキーワードを含む
- `SMALL` — 60文字未満・2行以内・箇条書きなし
- `MEDIUM` — それ以外（デフォルト）

### `createTask(prompt, discordUserId, taskId?, dangerLevel?, projectId?, taskType?)`

新しいタスクを作成して `data/tasks.json` に保存する。

**引数**

| 名前 | 型 | デフォルト | 説明 |
|------|----|-----------|------|
| `prompt` | `string` | 必須 | 依頼内容（500文字でスライス） |
| `discordUserId` | `string` | 必須 | 依頼した Discord ユーザー ID |
| `taskId` | `string \| null` | `null` | 指定 ID（省略時は `task_<timestamp>` 自動生成） |
| `dangerLevel` | `string` | `'低'` | 危険度（`'高'`/`'中'`/`'低'`） |
| `projectId` | `string` | `'default'` | プロジェクト ID（チャンネル名から決定） |
| `taskType` | `string \| null` | `null` | タスクタイプ（省略時は `IMPLEMENT`） |

**戻り値:** `Object` — 作成されたタスクオブジェクト

```js
// タスクオブジェクトの主要フィールド
{
  id:             string,      // タスク ID
  type:           string,      // タスクタイプ (IMPLEMENT 等)
  size:           string,      // タスクサイズ (SMALL/MEDIUM/LARGE)
  projectId:      string,      // プロジェクト ID
  prompt:         string,      // 依頼内容（〜500文字）
  state:          string,      // 現在の状態
  priority:       string,      // 優先度
  priorityReason: string,      // 優先度の理由
  dangerLevel:    string,      // 危険度
  assignee:       'Claude Code',
  requestedBy:    string,      // Discord ユーザー ID
  createdAt:      string,      // ISO 8601 タイムスタンプ
  updatedAt:      string,
  stateHistory:   Array,       // 状態変更ログ
  leaseOwner:     string|null, // リース占有中の worker ID
  leaseExpiresAt: string|null, // リース有効期限 (ISO 8601)
  leaseRole:      string|null, // Worker 役割
}
```

**例**
```js
const task = createTask('バグを修正する', 'user123', null, '中', 'myProject', 'FIX');
console.log(task.id);    // task_1234567890123
console.log(task.type);  // 'FIX'
console.log(task.size);  // 'SMALL' | 'MEDIUM' | 'LARGE'
```

### `updateState(taskId, newState, note?)`

タスクの状態を更新する。`DONE` に遷移すると月次アーカイブへ移動される。

**引数**

| 名前 | 型 | デフォルト | 説明 |
|------|----|-----------|------|
| `taskId` | `string` | 必須 | タスク ID |
| `newState` | `string` | 必須 | 新しい状態（`STATES` 定数から選択） |
| `note` | `string` | `''` | 状態変更の理由メモ |

**戻り値:** `Object | null` — 更新後のタスクオブジェクト

**副作用:**
- `IN_PROGRESS` 以外への遷移時はリースフィールドをクリア
- `DONE` 遷移時は `data/history/YYYY-MM.json` にアーカイブし `tasks.json` から除去
- 子タスクが `DONE` になった場合、親タスクの自動完了チェックを実行

### `claimNextTask(projectId, ownerId)`

**動作:** 優先度順にソートして最上位の PENDING タスクをクレーム。リースが有効期限内のタスクはスキップ。

## claude-runner.js

**役割:** Claude Code CLI をプログラムから起動して結果を返す。

### `run(prompt, workspacePath, projectRoot?, options?)`

Claude Code を非インタラクティブモードで実行する。

**引数**

| 名前 | 型 | デフォルト | 説明 |
|------|----|-----------|------|
| `prompt` | `string` | 必須 | Claude Code への指示文 |
| `workspacePath` | `string` | 必須 | タスク専用フォルダ（`prompt.md` / `result.md` 保存先） |
| `projectRoot` | `string \| null` | `null` | Claude Code の cwd（省略時は `workspacePath`）。AI_WORKER ルートを渡すと本体ファイルを編集可能 |
| `options` | `Object` | `{}` | `{ timeoutMs: number }` — タスクサイズ別タイムアウト |

**戻り値:** `Promise<{ output, duration, exitCode, costUsd, numTurns }>`

| フィールド | 型 | 説明 |
|------------|-----|------|
| `output` | `string` | Claude Code の出力テキスト（ANSI 除去済み） |
| `duration` | `number` | 実行秒数 |
| `exitCode` | `number` | プロセス終了コード |
| `costUsd` | `number \| null` | 実行コスト（$）。取得できない場合は `null` |
| `numTurns` | `number \| null` | モデル応答ターン数 |

**エラー:** タイムアウト・プロセス起動失敗・claude コマンド未インストール時に reject

**例**
```js
const { run } = require('./claude-runner');

const result = await run(
  'バグを修正してください',
  '/path/to/workspace/task_123',
  '/path/to/ai-worker',       // projectRoot
  { timeoutMs: 5 * 60 * 1000 }
);
console.log(result.output);   // 実行結果テキスト
console.log(result.costUsd);  // 例: 0.0042
```

**環境変数**

| 変数 | 説明 |
|------|------|
| `CLAUDE_COMMAND` | Claude CLI のパス（省略時: 自動解決） |
| `TASK_TIMEOUT_SECONDS` | デフォルトタイムアウト秒数（省略時: 300） |

## github.js

**役割:** タスク完了後に git add → commit → push を自動実行する。

### `commitAndPush(prompt, taskId)`

変更をコミットして GitHub に push する。

**引数**

| 名前 | 型 | 説明 |
|------|----|------|
| `prompt` | `string` | コミットメッセージの要約に使用 |
| `taskId` | `string` | タスク ID（コミットメッセージに記載） |

**戻り値:** `Promise<Object>`

```js
// 変更がない場合
{ skipped: true, reason: string }

// 実行した場合
{
  skipped:      false,
  subject:      string,   // コミット件名（例: "feat: [AI] xxx"）
  changedFiles: string[], // 変更ファイル一覧（git status --porcelain 形式）
  changedCount: number,
  pushed:       boolean,  // push 成功なら true
  pushError:    string|null, // push 失敗時のエラーメッセージ（機密マスク済み）
  repoPath:     string,
  branch:       string,
  repo:         string,   // "username/repo" 形式
}
```

**必要な環境変数**

| 変数 | 説明 |
|------|------|
| `ENABLE_GITHUB` | `true` の場合のみ push を実行 |
| `GITHUB_TOKEN` | GitHub Personal Access Token |
| `GITHUB_REPO` | リポジトリ名（例: `username/repo`） |
| `GITHUB_BRANCH` | push 先ブランチ（省略時: 現在のブランチ） |
| `GIT_REPO_PATH` | git リポジトリパス（省略時: AI_WORKER ルート） |

**セキュリティ:** トークンは HTTP Authorization ヘッダーで渡し、remote URL には埋め込まない。

## youtube-predictor.js

**役割:** YouTube 動画の視聴数バズ予測。ルールベース + ロジスティック回帰のハイブリッドモデル。  
**モデル保存先:** `data/youtube-model.json`

### `predict(video)`

動画データから視聴バズを予測する。

**引数:** `video: Object`

| フィールド | 型 | 説明 |
|------------|-----|------|
| `viewCount` | `number` | 再生数（0 または未設定で投稿前判定） |
| `likeCount` | `number` | いいね数 |
| `commentCount` | `number` | コメント数 |
| `title` | `string` | 動画タイトル |
| `description` | `string` | 動画説明文 |
| `tags` | `string[]` | タグ一覧 |
| `duration` | `number` | 動画尺（秒） |
| `publishedAt` | `string` | 投稿日時（ISO 8601） |
| `subscriberCount` | `number` | チャンネル登録者数 |
| `videoId` | `string` | YouTube 動画 ID（任意、ログ用） |

**戻り値:**
```js
{
  probability: number,      // バズ確率（0〜100 の整数）
  label:       string,      // 'hit' | 'miss' | 'unknown'
  confidence:  string,      // 'high' | 'medium' | 'low'
  buzzRatio:   number|null, // viewCount / subscriberCount（投稿前は null）
  usedML:      boolean,     // ML モデルを混合したか
  mlSamples:   number,      // 訓練サンプル数
  viewRange:   { low, median, high } | null, // 再生数レンジ推定
  benchmark:   { genreMedian } | null,       // ジャンル中央値ベンチマーク
  reasons:     Array,       // スコア主要因（上位3件）
}
```

**判定基準**
- `probability >= 60` → `label: 'hit'`
- `probability <= 40` → `label: 'miss'`
- それ以外 → `label: 'unknown'`

**投稿前モード:** `viewCount` が 0 または未設定の場合はルールベースのみ使用（登録者数・タイトル品質・タグ数・動画尺で採点）。

**例**
```js
const { predict } = require('./youtube-predictor');

const result = predict({
  viewCount:       10000,
  likeCount:       500,
  subscriberCount: 50000,
  title:           '【衝撃】〇〇してみた！',
  tags:            ['vlog', 'gaming', '挑戦'],
  duration:        600,
});
console.log(result.probability);  // 例: 72
console.log(result.label);        // 'hit'
console.log(result.reasons);      // [{ factor, detail, score }, ...]
```
