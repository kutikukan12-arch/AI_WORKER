## logger.js

`bot/utils/logger.js`

コンソールとファイル（`logs/YYYY-MM-DD.log`）への色付きログ出力。

### エクスポート

| 関数 | 説明 |
|------|------|
| `info(msg)` | 緑色・INFO レベル |
| `warn(msg)` | 黄色・WARN レベル |
| `error(msg)` | 赤色・ERROR レベル |
| `debug(msg)` | シアン・DEBUG レベル |

### 引数・戻り値

```
info(msg: string): void
warn(msg: string): void
error(msg: string): void
debug(msg: string): void
```

### 使用例

```js
const logger = require('./utils/logger');

logger.info('タスク処理を開始します');
logger.warn('ウェイトファイルが見つかりません');
logger.error(`タスク実行失敗: ${err.message}`);
logger.debug(`スコア計算結果: ${JSON.stringify(scores)}`);
```

## ai-predictor.js

`bot/utils/ai-predictor.js`

タスクの特徴量からルールベース＋ML ブレンドの予測を行うモジュール。

### predict()

全予測をまとめて返すメインエントリーポイント。

```
predict(
  prompt:   string,
  taskType: string = 'IMPLEMENT',
  taskSize: string = 'MEDIUM',
  output:   string = ''
): {
  routing: RoutingResult,
  outcome: OutcomeResult,
  time:    TimeResult,
  summary: string,
}
```

**引数**

| 引数 | 型 | 必須 | 説明 |
|------|----|------|------|
| `prompt` | string | ✓ | タスクの依頼文 |
| `taskType` | string | — | `'IMPLEMENT'` `'FIX'` `'RESEARCH'` `'DESIGN'` `'REVIEW'` `'DOCS'` `'OPS'` `'TEST'` |
| `taskSize` | string | — | `'SMALL'` `'MEDIUM'` `'LARGE'` |
| `output` | string | — | Claude Code の実行出力（AI ルーティング精度向上に使用） |

**戻り値**

```js
{
  routing: { recommended, confidence, scores, reason },  // predictAIRouting() の戻り値
  outcome: { probability, confidence, risks, bonuses },  // predictTaskOutcome() の戻り値
  time:    { estimateMin, estimateMax, unit },            // predictCompletionTime() の戻り値
  summary: string,                                        // Discord 表示用テキスト
}
```

**使用例**

```js
const { predict } = require('./utils/ai-predictor');

const result = predict('認証ミドルウェアのリファクタリング', 'REFACTOR', 'MEDIUM');
console.log(result.summary);
// → "🔮 **AI 予測モデル（初期バージョン）**\n..."
console.log(result.outcome.probability); // → 75 (%)
```

### predictAIRouting()

担当 AI を信頼スコア付きで予測する。

```
predictAIRouting(
  prompt:   string,
  taskType: string = 'IMPLEMENT',
  output:   string = ''
): {
  recommended: 'Claude Code' | 'Codex' | 'ChatGPT',
  confidence:  'high' | 'medium' | 'low',
  scores:      { 'Claude Code': number, 'Codex': number, 'ChatGPT': number },
  reason:      string,
}
```

**使用例**

```js
const { predictAIRouting } = require('./utils/ai-predictor');

const r = predictAIRouting('セキュリティ脆弱性を修正してください', 'FIX');
// r.recommended → 'Codex'
// r.confidence  → 'high'
// r.reason      → 'コード修正・最適化・セキュリティ改善のシグナルを検出'
```

### predictTaskOutcome()

タスクの成功確率（0〜100）を予測する。

```
predictTaskOutcome(
  prompt:   string,
  taskType: string = 'IMPLEMENT',
  taskSize: string = 'MEDIUM'
): {
  probability: number,    // 0〜100
  confidence:  'high' | 'medium' | 'low',
  risks:       string[],  // 検出したリスク要因ラベル
  bonuses:     string[],  // 検出したポジティブ要因ラベル
}
```

**リスク要因（penalty）**

| ラベル | ペナルティ |
|--------|-----------|
| 認証・機密関連 | -15 |
| データ削除操作 | -20 |
| 本番環境変更 | -10 |
| DB操作 | -8 |
| 大規模フェーズ | -5 |

**ポジティブ要因（bonus）**

| ラベル | ボーナス |
|--------|---------|
| テスト付き | +5 |
| バックアップ前提 | +3 |
| 最小変更 | +5 |
| ドキュメント系 | +3 |

**使用例**

```js
const { predictTaskOutcome } = require('./utils/ai-predictor');

const o = predictTaskOutcome('本番DBのマイグレーションを実行', 'OPS', 'LARGE');
// o.probability → 47  （本番環境 -10, DB操作 -8, LARGE -10 でペナルティが重なる）
// o.risks       → ['本番環境変更', 'DB操作']
// o.confidence  → 'medium'
```

### predictCompletionTime()

```
  taskSize: string = 'MEDIUM'

  estimateMax: number,   // 最長（分）
```

**TaskType 別の基本パラメータ**

| taskType | successBase | timeMin | timeMax |
|----------|------------|---------|---------|
| IMPLEMENT | 75% | 10分 | 60分 |
| FIX | 80% | 5分 | 30分 |
| RESEARCH | 90% | 5分 | 20分 |
| DESIGN | 85% | 5分 | 20分 |
| REVIEW | 92% | 3分 | 15分 |
| DOCS | 88% | 5分 | 25分 |
| OPS | 85% | 3分 | 10分 |
| TEST | 82% | 8分 | 30分 |

**使用例**

```js
const { predictCompletionTime } = require('./utils/ai-predictor');

const t = predictCompletionTime('IMPLEMENT', 'LARGE');
// t.estimateMin → 18  (10 * 1.8)
// t.estimateMax → 108 (60 * 1.8)
// t.unit        → 'minutes'
```

## youtube-predictor.js

`bot/utils/youtube-predictor.js`

YouTube 動画のヒット確率をルールベース＋ML ブレンドで予測する。

### VideoInput 型

```ts
interface VideoInput {
  title:           string;
  viewCount?:      number;   // 0 または省略 → 投稿前モード
  likeCount?:      number;
  commentCount?:   number;
  subscriberCount?: number;
  duration?:       number;   // 秒数
  tags?:           string[];
  description?:    string;
  publishedAt?:    string;   // ISO 8601
  videoId?:        string;
}
```

---

### predict()

動画データからヒット確率を予測する。

```
predict(video: VideoInput): {
  probability: number,           // 0〜100（%）
  label:       'hit' | 'miss' | 'unknown',
  confidence:  'high' | 'medium' | 'low',
  buzzRatio:   number | null,    // viewCount / subscriberCount
  usedML:      boolean,
  mlSamples:   number,
  viewRange:   { low: number, median: number, high: number } | null,
  benchmark:   { genreMedian: number } | null,
  reasons:     Reason[],
}
```

**Reason 型**

```ts
interface Reason {
  factor: string;              // 例: 'チャンネル規模', 'buzz_ratio'
  impact: 'positive' | 'negative';
  detail: string;              // 例: '登録者5万人（中規模）'
}
```

**判定ロジック**

- `viewCount` が 0 または未設定 → **投稿前モード**（タイトル・タグ・尺・登録者数で採点）
- `viewCount` が設定済み → **投稿後モード**（buzz_ratio・いいね率・コメント率で採点）
- buzz_ratio > 5.0 → `hit`、buzz_ratio < 0.3 → `miss`

**使用例**

```js
const { predict } = require('./utils/youtube-predictor');

// 投稿前モード
const result = predict({
  title:           '【保存版】Python入門 完全ガイド2025',
  subscriberCount: 50000,
  duration:        720,
  tags:            ['Python', 'プログラミング', '入門', '初心者', '2025'],
});
// result.probability → 62
// result.label       → 'hit'
// result.reasons[0]  → { factor: 'チャンネル規模', impact: 'positive', detail: '登録者5万人（中規模）' }

// 投稿後モード
const r2 = predict({
  title:           'daily vlog',
  subscriberCount: 1000,
  viewCount:       8500,
  likeCount:       300,
  commentCount:    40,
  duration:        900,
});
// r2.buzzRatio → 8.5  → hit
```

---

### train()

ラベル付きサンプルから ML モデルをバッチ学習し `data/youtube-model.json` に保存する。

```
train(samples: (VideoInput & { label: 'hit' | 'miss' })[]): {
  weights:             number[],
  sampleCount:         number,
  hitCount:            number,
  missCount:           number,
  trainDirectionalAcc: number,  // 0〜1（学習データ上での方向性正解率）
  trainedAt:           string,
} | null
```

- サンプル数が 10 件未満の場合は `null` を返す

**使用例**

```js
const { train } = require('./utils/youtube-predictor');

const result = train([
  { title: 'A', subscriberCount: 10000, viewCount: 80000, label: 'hit' },
  { title: 'B', subscriberCount: 10000, viewCount: 500,   label: 'miss' },
  // ... 最低 10 件以上
]);
if (result) {
  console.log(`学習完了: ${result.sampleCount}件 精度:${(result.trainDirectionalAcc * 100).toFixed(1)}%`);
}
```

### getModelStatus()

現在の ML モデルの状態を返す。

```
getModelStatus(): {
  trained:             boolean,
  sampleCount:         number,
  hitCount?:           number,
  missCount?:          number,
  trainDirectionalAcc?: number | null,
  trainedAt?:          string,
}
```

**使用例**

```js
const { getModelStatus } = require('./utils/youtube-predictor');

const s = getModelStatus();

  console.log(`モデル学習済み: ${s.sampleCount}件 (${s.trainedAt})`);
} else {
  console.log('モデル未学習');
}
```

## predict-cli.js（CLI）

`bot/predict-cli.js`

コマンドライン から YouTube ヒット予測を実行するスクリプト。

### 基本構文

```
node bot/predict-cli.js --title "タイトル" --subs <登録者数> [options]
```

### オプション一覧

| フラグ | 省略形 | 型 | 説明 |
|--------|--------|----|------|
| `--title` | `-t` | string | 動画タイトル **（必須）** |
| `--channel` | `-c` | string | チャンネル名（表示用のみ） |
| `--subs` | `-s` | number | チャンネル登録者数 |
| `--views` | `-v` | number | 視聴数（0 または省略 → 投稿前モード） |
| `--likes` | `-l` | number | いいね数 |
| `--comments` | — | number | コメント数 |
| `--duration` | `-d` | number | 動画長（秒、例: `600` = 10分） |
| `--tags` | — | string | タグ（カンマ区切り、例: `"AI,技術,解説"`） |
| `--tag` | — | string | タグ（複数回指定可、例: `--tag AI --tag 技術`） |
| `--desc` | — | string | 概要欄テキスト |
| `--published` | — | string | 投稿日時 ISO 8601（例: `2024-03-01T18:00:00Z`） |
| `--json` | — | flag | JSON 形式でも出力する |
| `--help` | `-h` | flag | ヘルプを表示して終了 |

### 使用例

```bash
# 投稿前予測（viewCount 未指定）
node bot/predict-cli.js \
  -t "【衝撃】AIが仕事を奪う未来" \
  -s 50000 \
  --tags "AI,テクノロジー,未来" \
  -d 600

# 投稿後予測（viewCount 指定）
node bot/predict-cli.js \
  -t "daily vlog" \
  -s 1000 \
  -v 8500 \
  -l 300 \
  --comments 40 \
  -d 900

# JSON 出力付き
node bot/predict-cli.js -t "解説動画" -s 20000 --json
```

### 終了コード

| コード | 意味 |
|--------|------|
| `0` | 正常終了 |
| `1` | 入力エラーまたは予測処理失敗 |

## task-manager.js

`bot/utils/task-manager.js`

タスクの CRUD・状態管理・優先度・リース制御を担うモジュール。タスクは `data/tasks.json` に保存される。

### 定数

```js
const { STATES, TASK_TYPES, TASK_SIZES } = require('./utils/task-manager');

STATES.PENDING       // '未着手'
STATES.IN_PROGRESS   // '作業中'
STATES.REVIEWING     // 'レビュー待ち'
STATES.AWAITING      // '人間確認待ち'
STATES.DONE          // '完了'
STATES.ON_HOLD       // '保留'

TASK_TYPES.IMPLEMENT // 'IMPLEMENT' — コード実装・機能追加
TASK_TYPES.FIX       // 'FIX'       — バグ修正
TASK_TYPES.REFACTOR  // 'REFACTOR'
TASK_TYPES.RESEARCH  // 'RESEARCH'
TASK_TYPES.DOCS      // 'DOCS'
TASK_TYPES.TEST      // 'TEST'
TASK_TYPES.REVIEW    // 'REVIEW'

TASK_SIZES.SMALL     // 'SMALL'  — 30分以内
TASK_SIZES.MEDIUM    // 'MEDIUM' — 半日以内（デフォルト）
TASK_SIZES.LARGE     // 'LARGE'  — 複数日・分割推奨
```

### createTask()

タスクを新規作成して `data/tasks.json` に保存する。

```
createTask(
  prompt:        string,
  discordUserId: string,
  taskId?:       string | null,    // null → `task_${Date.now()}` を自動生成
  dangerLevel?:  '高' | '中' | '低',
  projectId?:    string,           // デフォルト: 'default'
  taskType?:     string | null     // TASK_TYPES のいずれか。null → 'IMPLEMENT'
): Task
```

**戻り値（Task オブジェクト）**

```ts
interface Task {
  id:             string;
  type:           string;       // TASK_TYPES のいずれか
  size:           string;       // TASK_SIZES のいずれか（prompt から自動推定）
  projectId:      string;
  prompt:         string;       // 最大 500 文字にトリミング
  state:          string;       // STATES のいずれか
  priority:       string;
  priorityReason: string;
  dangerLevel:    string;
  assignee:       string;       // 'Claude Code'
  requestedBy:    string;
  createdAt:      string;       // ISO 8601
  updatedAt:      string;
  stateHistory:   StateEntry[];
  reviewResult:   any;
  codexResult:    any;
  prUrl:          string | null;
  notes:          string;
  rootTaskId:     string | null;
  childTasks:     string[];
  timeoutCount:   number;
  splitCount:     number;
  lastError:      string | null;
  errorType:      string | null;
  leaseOwner:     string | null;
  leaseExpiresAt: string | null;
  leaseRole:      string | null;
}
```

**使用例**

```js
const { createTask } = require('./utils/task-manager');

const task = createTask(
  '認証エラーを修正してください',
  'U12345678',
  null,     // taskId 自動生成
  '高',
  'ai-worker',
  'FIX'
);
console.log(task.id);    // → 'task_1717600000000'
console.log(task.size);  // → 'SMALL' または 'MEDIUM' (prompt 長から自動推定)
```

### claimNextTask()

### estimateTaskSize()

prompt の内容からタスクサイズを自動推定する。

```
estimateTaskSize(prompt: string): 'SMALL' | 'MEDIUM' | 'LARGE'
```

**判定基準**

| サイズ | 条件 |
|--------|------|

**使用例**

```js
const { estimateTaskSize } = require('./utils/task-manager');
```

### commitAndPush()

変更をステージング・コミット・プッシュする。`ENABLE_GITHUB=true` かつ `GITHUB_TOKEN` が設定されている場合のみプッシュを実行する。

```
commitAndPush(
  message?:  string,  // コミットメッセージ（省略時: generateCommitMessage() で自動生成）
  taskId?:   string   // ログ・エラーレポートに使用するタスクID

```

**使用例**

```js
const { commitAndPush } = require('./utils/github');

const r = await commitAndPush('fix: 認証エラーを修正', 'task_1717600000000');
if (r.success) {
  console.log(`コミット完了: ${r.commitHash}`);
} else {
  console.error(`失敗: ${r.message}`);
}
```

### generateCommitMessage()

変更ファイル一覧から Conventional Commits 形式のコミットメッセージを自動生成する。

```
generateCommitMessage(files?: string[]): string
```

**使用例**

```js
const { generateCommitMessage } = require('./utils/github');

const msg = generateCommitMessage(['bot/utils/ai-predictor.js', 'tests/predictor.test.js']);
// → 'feat: update ai-predictor.js, predictor.test.js'
```

### maskSecret()

文字列からトークン・秘密鍵・JWT 等の機密情報をマスクする。内部では `redact.js#redact` へ委譲する。

```
maskSecret(str: string): string
```

**使用例**

```js
const { maskSecret } = require('./utils/github');

maskSecret('token=ghp_abc123xyz');
// → 'token=[REDACTED]'
```
