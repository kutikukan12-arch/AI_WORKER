## 0. このテストの目的（最重要・誤解防止）

> **いま検証するのは「売れるか」ではなく「使われるか・また使いたくなるか」＝需要の有無。**

- 売る判断・価格決定はしない。**課金は仮説の聞き取り（調査）まで**。
- 「正しく動くか（技術）」は守谷CTOが担保済み前提。CSが見るのは **利用者の体感と本音**。
- 評価は点数より **「どこで詰まったか」「どの瞬間に価値を感じたか」の観察と発言の記録**が主役。

### テストの基本ルール
- **被験者を助けない。** 迷っても口を出さず「迷った事実」を記録する（初回体験の最大の情報源）。
- **声に出してもらう（Think Aloud）。** 操作中に考えていることを実況してもらう。
- **誘導しない。** 「便利ですよね？」ではなく「いま何を考えましたか？」と中立に聞く。
- 1人15〜20分。最低 **5〜6人**（できればVTuber/実況/切り抜き等ターゲット属性を含む）。

## ★ β核 価値チェック（最優先・精度より上位）

> この商品の価値は「未来を100%予測すること」ではなく、**投稿前の失敗確率を下げ、改善できる行動を示すこと**。点数の当たり外れより、**下の3つがYESか**を最優先で見る。

| 価値項目 | 確認 | 記録（YES/△/NO ＋ 発言） |
|---|---|---|
| V-1 次の行動が分かる | 診断後、「次に何を直すか」を具体的に言えたか | |
| V-2 不安が減る | 「これで伸びるか不安」が診断後に軽くなったか（表情・発言で確認） | |
| V-3 また使いたい | 次の投稿前にも使いたいと思ったか | |

▶ 質問例：「診断を見て、次に何をしますか？」「やる前より、少し安心できました？」「次に動画を出すとき、また開きそうですか？」
▶ 判定：V-1〜V-3 が揃えば、スコアが多少ズレていても**商品価値は成立**。逆に揃わなければ、精度が高くても価値なし。

## ⑤ 課金可能性（**調査のみ・売り込み禁止**）

> 価格を提示して反応を見るのではなく、**価値と支払い動機の有無を聞き取る**。

| 観点 | 聞く | 記録 |
|---|---|---|
| 5-1 無料なら | 無料なら使い続けるか | |
| 5-2 支払い動機 | お金を払うとしたら**どんな機能・状況なら**払うか | |
| 5-3 有料化条件 | 有料にするなら最低何が必要だと思うか | |
| 5-4 価値の源泉 | この診断の何に一番価値を感じたか（自由記述） | |
| 5-5 不要機能 | 逆に「これは要らない」と思った要素 | |

▶ 中立質問（誘導しない）：「もしこれが有料だとしたら、どう感じますか？」→ 価格はこちらから言わない。被験者が額を口にしたら記録。

## 4. 全体集計（PM/経営に渡す need-validation サマリー）

| KPI | 目標感（需要の目安） | 実測 |
|---|---|---|
| **★ V-1 次の行動が分かった率** | **最優先・精度より上位** | |
| **★ V-2 不安が減った率** | **最優先・精度より上位** | |
| **★ V-3 また使いたい率** | **最優先・精度より上位** | |
| タスク完了率（自力で診断完了） | 80%以上 | |
| 初回詰まりゼロ率 | 過半数 | |
| 「実データで診断された」と感じた率 | 高いほど商品の核が伝わっている | |
| **自発的な再診断率** | **最重要シグナル** | |
| リピート意向「強い/ある」率 | 過半数なら需要あり | |
| 課金の芽が出た人数 | （調査値・参考） | |
| 禁止表現の出現件数 | **0でなければ即修正** | |

### 需要判定の目安（PM向け）
> まず **価値基準 V-1〜V-3（次の行動が分かる／不安が減る／また使いたい）** を見る。精度は二の次。
- 🟢 **需要あり**：V-1〜V-3が過半数YES＋自発的再診断が起きる → Phase2（課金設計）検討へ
- 🟡 **要改善**：価値の方向は感じるが V のどれかが弱い／詰まり・曖昧さが多い → UX・根拠表示を改善して再テスト
- 🔴 **需要薄**：V-1〜V-3が揃わず再利用意向も低い → コンセプト見直し（※精度が高くてもVが揃わなければここ）

## 5. 注意・運用メモ
- このフェーズは**需要確認**。価格決定・販売開始の判断材料ではない。
- 禁止表現（再生数予測/レンジ/保証）を発見したら、点数に関わらず**最優先で開発へ差し戻し**。
- 生の発言ログ（原文）は宝。要約しすぎず残す。
- 結果は相沢CS → 市川PM へ。PMが need-validation サマリーにまとめ、金森CFOコスト確認と合わせて経営判断材料にする。
